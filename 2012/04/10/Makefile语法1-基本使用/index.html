Makefile 语法 一

发表于 2012-04-10 |||| 阅读次数

#1简介

Linux 环境下的程序员如果不会使用 GNU make 来构建和管理自己的工程，应该
不能算是一个合格的专业程序员，至少不能称得上是 Unix 程序员。在 Linux（unix）环
境下使用 GNU 的 make 工具能够比较容易的构建一个属于你自己的工程，整个工程的
编译只需要一个命令就可以完成编译、连接以至于最后的执行。不过这需要我们投入一
些时间去完成一个或者多个称之为 Makefile 文件的编写。此文件正是 make 正常工作
的基础。Makefile只能在Linux下使用。

####编译整个工程你所要做的唯一的一件事就是在 shell 提示符下输入 make 命令。

####make 在执行时，需要一个命名为 Makefile 的文件。也就是说你要有个文件叫Makefile 其它什么都不行。

下面是一个Makefile文件的内容：

Makefile

#mainapp 是 目标文件 编译这个mainapp文件 需要有后面的文件
mainapp:main.o plus.o minus.o multi.o divi.o
    gcc main.o plus.o minus.o multi.o divi.o -o mainapp
#而要得到main.o这个文件需要main.c 这个文件
mian.o:main.c
#而main.c 要执行一个gcc的编译命令才能生成main.o  那么一个tab键就是执行命令的意思。可以执行shell命令 一个命令一行
    gcc -c main.c
plus.o:plus.c
    gcc -c plus.c
minus.o:minus.c
    gcc -c minus.c
multi.o:multi.c
    gcc -c multi.c
divi.o:divi.c
    gcc -c divi.c
clean:
    rm -f *.o

###$>执行make命令运行结果 生成各个的.o 文件和 mainapp这个可执行文件

###$>执行./mainapp 运算结果

###$>执行make clean 清楚生成的.o文件

###$>如果已经生成了mainapp文件，再执行make 是无法再次生成的 要对某个文件修改才能再次编译，比如执行touch main.c 在执行make就可以再次编译。

#2.优化

这样写起来并不简单啊 ，是否可以优化？

在实际工作中大家都比较认同的方法是，使用一个变量 “objects”、“OBJECTS”、“objs”、“OBJS”、“obj”或者“OBJ”来作为所有的.o 文 件的列表的替代。在使用到这些文件列表的地方，使用此变量来代替

上面的文件内容可改为：

#object 是变量
objects=main.o plus.o minus.o multi.o divi.o
mainapp:$(objects)
    gcc $(objects) -o mainapp
#%.o 所有的输出文件  %.c所有的依赖文件
%.o:%.c
#$^ 输出文件 $@依赖文件
    gcc -c $^ -o $@
clean:
    rm -f *.o

#3.再优化

#一般我们可以使用“$(wildcard *.c)”来获取工作目录下的所有的.c 文件列表
source=$(wildcard *.c)
#；可以使用“$(patsubst %.c,%.o,$(wildcard *.c))”，首先使用“wildcard” 函数获取工作目录下的.c 文件列表；之后将列表中所有文件名的后缀.c 替换为.o。这样 我们就可以得到在当前目录可生成的.o 文件列表
objects=$(patsubst %.c,%.o,$(source))
mainapp:$(objects)
    gcc $^ -o $@
%.o:%.c
    gcc -c $^ -o $@
clean:
    rm -f *.o
