建造者模式

发表于 2014-09-07 | | 阅读次数:

###标准构建者模式

  • 定义

将一个复杂的构建与他的表示分离，使得同样的构建过程可以创建不同的表示。

  • 角色

    • 抽象建造者

    • 具体建造者

    • 设计者

    • 产品

    eg：比如要建一个房子，房子就是产品，设计者就是设计房子的，具体建造者就是工人，而工人按照先打地基，后垒墙，最后封顶的顺序，这个顺序就是抽象建造者。

    • 抽象建造者
public interface IBuild {
    void makeRoot();
    void makeWall();
    void makeTop();
    void makeWindows();
    Room build();
}
  • 具体建造者,持有對產品的引用.具体的建造者持有对产品的引用，这个原则很重要，咋一看没什么，其实写代码中就会发现，在具体建造者中要有产品的对象实例。 
public class LeequerRoomBuild implements IBuild {
    Room mRoom = new LeequerRoom();

    @Override
    public void makeRoot() {
        System.out.println("正在打地基");
        mRoom.setRoot("地基");
    }

    @Override
    public void makeWall() {
        System.out.println("正在建墙面");
        mRoom.setWall("墙");
    }

    @Override
    public void makeTop() {
        System.out.println("正在建房顶");
        mRoom.setTop("房顶");
    }

    @Override
    public Room build() {
        makeRoot();
        makeWall();
        makeTop();
        return mRoom;
    }

    @Override
    public void makeWindows() {
        System.out.println("正在建窗戶");

    }


}

以上是标准的建造者模式，在编写的过程中，发现，在具体构建者中build方法和设计者中的buildRoom方法可以去掉这记者的这个方法，而设计者去掉这个方法，就没有存在的意义了。所以这就是建造者模式的改进型。

###建造者模式的改进型

  • 产品

    public class LeequerRoom {
    private String root;
    private String top;
    private String wall;
    private String window;
    public void applyRoom(RoomParam param) {
        root = param.root;
        top = param.top;
        wall = param.wall;
        window = param.window;
    }

    @Override
    public String toString() {
        return "leequer 房子:" + root + ":" + wall + ":" + top+":"+window;
    }
}

    我们为了成员变量的安全，不能随意改动，删除了get，set方法，但这样谁也无法改变它了，RoomParam 是建造者中的内部类，就是用来解决成员变量无法访问的问题。

  • 具体建造者

      public class LeequerRoomBuild  {
    LeequerRoom mRoom = new LeequerRoom();
    RoomParam mRoomParam =  new RoomParam();
    public LeequerRoomBuild makeRoot() {
        System.out.println("正在打地基");
        mRoomParam.root = "地基";
        return this;
    }

    public LeequerRoomBuild makeWall() {
        System.out.println("正在建墙面");
        mRoomParam.wall = "墙";
        return this;
    }

    public LeequerRoomBuild makeTop() {
        System.out.println("正在建房顶");
        mRoomParam.top = "屋顶";
        return this;
    }

    public LeequerRoom build() {
        mRoom.applyRoom(mRoomParam);
        return mRoom;
    }

    public LeequerRoomBuild makeWindows() {
        System.out.println("正在建窗戶");
        mRoomParam.window="窗户";
        return this;
    }

    class RoomParam {
        public String root;
        public String top;
        public String wall;
        public String window;
    }
}

具体建造者中添加内部类，在build方法中调用产品的applyRoom方法，给产品赋值，也就是说，在没有调用build的方法之前，所有的值的修改都没有赋值。都是可以再次修改的。这样做，隐藏了构建过程。

  • 建造者模式的优点

无须关心产品对象的具体组装过程，只需指定具体建造者的类型即可。

建造者模式与抽象工厂模式有点相似，但是建造者模式返回一个完整的复杂产品，而抽象工厂模式返回一系列相关的产品；在抽象工厂模式中，客户端通过选择具体工厂来生成所需对象，而在建造者模式中，侧重于一步步构造一个复杂对象，然后将结果返回。

如果将抽象工厂模式看成一个汽车配件生产厂，生成不同类型的汽车配件，那么建造者模式就是一个汽车组装厂，通过对配件进行组装返回一辆完整的汽车。
