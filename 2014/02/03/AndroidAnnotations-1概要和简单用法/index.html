AndroidAnnotations-1 概要和简单用法

发表于 2014-02-03   |    阅读次数

AndroidAnnotations主要用于简化Android开发，项目地址，最直接的感觉就是不用写太多的findviewbyid了。其实在AFinal中也有类似的功能。

androidstudio的gradle的androidannotations的配置就不说了。直接看代码应用。

//在类的上面使用@EActivity 来实现原来setContent的方法
@EActivity(R.layout.translate) // Sets content view to R.layout.translate
public class TranslateActivity extends Activity {

    // Injects R.id.textInput 不用再写findviewby(R.id.textInput)
    @ViewById 
    EditText textInput;

    @ViewById(R.id.myTextView) // Injects R.id.myTextView
    TextView result;
        //资源文件的使用
    @AnimationRes // Injects android.R.anim.fade_in
    Animation fadeIn;

        //点击事件
    @Click // When R.id.doTranslate button is clicked 
    void doTranslate() {
     translateInBackground(textInput.getText().toString());
    }
        //此方法在后台运行
    @Background // Executed in a background thread
    void translateInBackground(String textToTranslate) {
     String translatedText = callGoogleTranslate(textToTranslate);
     showResult(translatedText);
    }
        //此方法在ui线程运行
    @UiThread // Executed in the ui thread
    void showResult(String translatedText) {
         result.setText(translatedText);
         result.startAnimation(fadeIn);
    }

// [...]
}

开发中遇到的问题

  • 由于用到了注解，在AndroidManifest.xml中设置activity，不能用原来的类名，需要加下划线，类似这样：

    <activity
    android:name=".TranslateActivity_”/>

  • 还有就是原来集成activity的方法基本都不能用了，原因是当你 使用了Annotations，框架会帮你生产一个你的类名对应的 类名_ 文件，相当于你的原始的Activity集成了这个带下划线的类，所以声明周期你都不能像原来那样写了.举个栗子：

    @Override
public void onBackPressed() {
            showAlertDialog();
    super.onBackPressed();
}

    你会发现这个方法执行了，但是如果你在这个方法中弹出一个是否退出的选择框，运行结果也会有对话框弹出，但是这个选择框不会等你选择，就直接跳过了。你需要这样写：

      public void onBackPressed() {
showAlertDialog();
}

    可以试试 onResume onStop方法 。

  • 还有，你的onCreate方法最好还是不用了，Annotations有个注解代替了它。

          @AfterViews
  void initView() {

}

    @AfterViews 表示view都初始化结束后的调用的方法，initView是可以方法名字，你可以自己定义其它名字，这里写原来onCreate的内容。就不会报空指针了。
