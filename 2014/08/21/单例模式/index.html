LS2-设计模式

单例模式

饿汉模式

private static Singleton instance = new Singleton();

静态实例，类加载的时候就把实例加载到内存中。线程安全，但消耗大。

懒汉模式

class Singleton {

    private static Singleton instance = null;      

    public  static   Singleton instance() {      
        if (instance == null) {      
            instance = new Singleton();      
        }      
        return instance;     
    }     
}

存在线程安全问题。若多个线程，同时访问这个方法，当第一个线程判断到instant==null去new一个实例的时候，另一个线程执行到判断instant是否为空，这是，还没new成功，那第二个线程就得到也是实例为空，就去实例化，这样就会有两个实例。解决这个问题就用线程同步：
class Singleton {

    private static Singleton instance = null;      

    public synchronized static   Singleton instance() {      
        if (instance == null) {      
            instance = new Singleton();      
        }      
        return instance;     
    }     
}

这里有一个性能问题：多个线程同时访问这个方法时，会因为同步而导致每次只有一个线程运行，影响程序性能。

DCL （double check Lock）

class Singleton {   

      private static Singleton instance = null ;   

      public static Singleton instance() {   
          if (instance == null ) {
              synchronized (this) {   
                  if (instance == null)
                     instance = new Singleton();
             }
         }
         return instance;
     }
}

这个解决方法在jdk1.5后，jvm 指令集乱序问题，导致有可能这样也不安全。解决方法是加volatile。使指令集不乱序。 

class Singleton {   

      private volatile static Singleton instance = null ;   

      public static Singleton instance() {   
          if (instance == null ) {
              synchronized (this) {   
                  if (instance == null)
                     instance = new Singleton();
             }
         }
         return instance;
     }
}

##面试题

Q：引用父类的静态字段会不会初始化子类？

A:不会，（解释：子类点父类的静态字段，无法获取）

Q:通过自定义对象数组类型 会不会加载该类型？
Class _class [] = {Person.class,Cat.class}
这样声明的class数组，数组中的元素Person和Cat类是否会被加载？

A:不会

原因：通过new和反射Class.from()可以初始化一个类,其它的都不行。java机制。

Q:Person{ public static int a=0；。。。} 调用Person.a的时候会初始化Person类么？

A:no 静态变量是在静态块中的，在class字节码加载的时候就加载到静态块中了。
