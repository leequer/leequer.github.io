—————–Git———-

文件如果之前已经被git跟踪了，这样修改是没有用的。每次修改完以后，用git status还是能看到提示它被修改了。

解决办法就是在git中删除这个文件的跟踪记录，用这个命令

git rm –cached 文件名路径和名称

这样就从git的跟踪记录中删除了这个文件的跟踪记录。配合之前在.gitignore加的那行配置，以后你修改 project.iws 这个文件，git就不会有提示了。

不想继续追踪某个文件

git update-index –assume-unchanged build.gradle

想再次继续跟踪某个文件

git update-index –no-assume-unchanged your_file_path

当你修改了一些文件，但还不能提交，于是需要保持当前的修改，先合并

git stash

打开保持的修改

git stash pop
