Rxjava-4 基本用法—Subscriber(订阅)其它方法 

正如前面所提到的，Observer 和 Subscriber 具有相同的角色，
而且Observer 在 subscribe() 过程中最终会被转换成 Subscriber 对象，
因此，从这里开始，后面的描述将用 Subscriber 来代替 Observer ，这样更加严谨。
  • 本文介绍的方法是和subscribe(Observer) 和 subscribe(Subscriber)相同功能的方法。也就是订阅这个动作的其它方法。这两个方法是订阅观察者对象。

  • 订阅者还可以使用以下不完整的定义回调。

    /**
*Action0 是 RxJava 的一个接口，它只有一个方法 call()，这个方法是无参无返回值的；

由于 onCompleted() 方法也是无参无返回值的，因此 Action0 可以被当成一个包装对象，将 onCompleted() 的内容打包起来将自己作为一个参数传入 subscribe() 以实现不完整定义的回调。

这样其实也可以看做将 onCompleted() 方法作为参数传进了 subscribe()，相当于其他某些语言中的『闭包』。 

其实我理解就是Action0就是实现了原来观察者的onCompleted方法。当被观察者发送结束订阅的事件的时候，这个Action0就执行。
*/
   Action0 onCompletedAction = new Action0() {
    //onCompleted();
    @Override
    public void call() {
           System.out.println("Action0->call");
    }
};

/**
* 此处实现了原来的// onNext()方法
*/
Action1<String> onNextAction = new Action1<String>() {

@Override
        public void call(String s) {
            Log.d(tag, s);
    }
};
/**
*此处实现了原来的额onError方法
*/
Action1<Throwable> onErrorAction = new Action1<Throwable>()             {
    // onError()
    @Override
    public void call(Throwable throwable) {
        // Error handling
    }    
};
  • 以上三个对象实现了观察者的三个响应方法
  • 把被观察者和观察者建立关系。

// 自动创建 Subscriber ，并使用 onNextAction 来定义 onNext().

observable.subscribe(onNextAction);

// 自动创建 Subscriber ，并使用 onNextAction 和 onErrorAction 来定义 onNext() 和 onError(),如果没有异常 onErrorAction中的call方法就不会被执行。

observable.subscribe(onNextAction, onErrorAction);

// 自动创建 Subscriber ，并使用 onNextAction、 onErrorAction 和 onCompletedAction 来定义 onNext()、 onError() 和 onCompleted()，如果没有异常 onErrorAction中的方法就不会执行。在没有异常情况下onCompletedAction方法会执行。

observable.subscribe(onNextAction,onErrorAction,onCompletedAction);

全部代码：

public class ActionTest {
//被观察者发送三个事件给观察者
Observable observable = Observable.just("hello", "hi", "123");

Action0 onCompletedAction = new Action0() {
    //onCompleted();
    @Override
    public void call() {
        System.out.println("Action0->call");
    }
};

Action1<String> onNextAction = new Action1<String>() {
    // onNext()
    @Override
    public void call(String s) {
        System.out.println("Action1-->call---->"+s);
    }
};
Action1<Throwable> onErrorAction = new Action1<Throwable>() {
    // onError()
    @Override
    public void call(Throwable throwable) {
        System.out.println("Action1-->call-->"+throwable.getMessage());
    }
};

public static void main (String [] arg)
{
    ActionTest _actionTest = new ActionTest();
    _actionTest.observable.subscribe(_actionTest.onNextAction,_actionTest.onErrorAction,_actionTest.onCompletedAction);
}
}
