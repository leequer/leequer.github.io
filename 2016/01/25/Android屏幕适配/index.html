Android屏幕适配资料

发表于 2016-01-25 | 阅读次数

引言

  • 屏幕适配的原因就不说了，都是泪水。说说策略吧。我们应该适配什么样的屏幕。根据大数据统计结果，市面上的android设备屏幕分辨率占比最高的是
    480x800的，接下来是320x480的，720X1280，480x854，540x960，1080x1920。适配了这六种屏幕，基本就可以适配大部分手机了。

  • 当然你如果适配平板，那就另说了。

重要概念

做适配之前，需要了解几个概念。

  • 屏幕尺寸：屏幕尺寸就是屏幕对角线的长度，1英寸=2.54cm，常见的有2.4，2.8，3.5，3.7，4.2，5.0，5.5，6.0等
  • 屏幕分辨率：是指在屏幕横向和纵向上的点数。单位是px。1px=一个像素点。一般表示为纵像素横像素 eg：1280720，也就是纵向排列有1280个点。
  • 屏幕像素密度dpi：每英寸上的像素点的多少。单位是dpi（dot per inch）。单一条件不变情况下，屏幕屏幕越小，分辨率越大，像素密度就越大。

  • dp dip dpi ：dp和dip相同，Density independent pixels的缩写，既和密度无关像素。

    一寸中的像素点是160个的话，这个屏幕的像素密度就160dpi。
在android中，以160dpi为标准，1px=1dip=1dp 
也就说如果屏幕一英尺里面有160个像素点的话，此时的dp=px。
如果屏幕像素密度是320dpi，也就是说一个英寸里面有320个像素点的话，1dp=2px，这样在android中设置了1dp，那就会在屏幕上占据2个像素点的距离，这样在分辨率不同的设备上可以显示大致相同的比例的图像。
这就是写layout布局时要写dp的原因。

//dp 转 px 转换代码
public static int dip2px(Context context, float dpValue) {
    final float scale = context.getResources().getDisplayMetrics().density;
    return (int) (dpValue * scale + 0.5f);
}
按照上面dip转px的算法，dpvalue 传1 的话，LG G3手机的结果返回是4，也就是1dp=4px。那就是手机的像素密度是160*4=640dpi。也就是一英寸有640个点。
//px转dp代码
public static int px2dip(Context context, float pxValue) {
    final float scale = context.getResources().getDisplayMetrics().density;
    return (int) (pxValue / scale + 0.5f);
    }

px转dp ，按这个算下来，我们就可以更具美工给的px长度换算应该在layout中写多少dp了。
