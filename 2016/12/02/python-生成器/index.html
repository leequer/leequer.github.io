'''
生成器 generator：真正的赋值推迟到使用时。
避免开辟大的内存，只使用部分内容

如果列表元素可以按照某种算法推算出来，
那我们是否可以在循环的过程中不断推算出后续的元素呢？
这样就不必创建完整的list，从而节省大量的空间。
在Python中，这种一边循环一边计算的机制，称为生成器（Generator）
通过next 来获得元素

因为要推算出下个是什么，所以 必须从第一个开始遍历
'''
# 先创建一个包含100个数字的集合，然后只取前10个。浪费资源
square_table = []
for i in range(100):
    square_table.append(i * i)
for i in range(5):
    print(square_table[i])

# 用生成器避免这种浪费
# 列表推导时用()替换[]
square_generator = (x * x for x in range(100))
print(type(square_generator))
for i in range(5):
    print(next(square_generator))

'''
迭代器
  可以直接作用于for循环的对象统称为可迭代对象：Iterable
  可以被next()函数调用并不断返回下一个值的对象称为迭代器：
Iterator（表示一个惰性计算的序列）,可以惰性计算的才是迭代器
  集合数据类型如list、dict、str等是Iterable(可迭代的对象)
但不是Iterator（迭代器），不过可以通过iter()函数获得一个Iterator对象



'''

from collections import Iterable
from collections import Iterator

print(isinstance([1, 2, 3], Iterable))
print(isinstance({}, Iterable))
print(isinstance(123, Iterable))
print(isinstance('abc', Iterable))

print(isinstance([1, 2, 3], Iterator))

g = (x * x for x in range(10))
print("g is ", type(g))
print("生成器是可迭代的么？", isinstance(g, Iterable))
print('生成器是一个迭代器么？', isinstance(g, Iterator))
for i in g:
    print(i)
