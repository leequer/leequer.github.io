简介

Truffle是以太坊智能合约的开发环境，测试框架，以太坊的资源管理通道，致力于让以太坊上的开发变得简单。

Truffle有以下特点：

  • 内置的智能合约编译，链接，部署和二进制文件的管理。
  • 快速开发下的自动合约测试。
  • 脚本化的，可扩展的部署与发布框架。
  • 部署到不管多少的公网或私网的网络环境管理功能
  • 使用EthPM&NPM提供的包管理，使用ERC190标准。
  • 与合约直接通信的直接交互控制台（写完合约就可以命令行里验证了）。
  • 可配的构建流程，支持紧密集成。
  • 在Truffle环境里支持执行外部的脚本。

truffle使用

前置工作

环境要求 NodeJS 5.0+

先安装node，依次执行以下2条命令： 

curl -sL https://deb.nodesource.com/setup_9.x 

sudo -E bash apt-get install nodejs

truffle安装

安装方式： 

$ npm install -g truffle

TestRPC客户端安装

Truffle需要以太坊客户端，需要支持标准的JSON RPC API。用普通话说就是 需要一个可以跑合约的以太坊私链。就是以太坊-私有链的安装，启动，操作一文中我们建立的私链，启动这个私链也可以。

安装TestRPC为例: 

sudo -E npm install ethereumjs-testrpc -g

运行下面命令启动TestRPC

testrpc

可看到testrpc版本号、初始化以太坊地址和监听端口，此处版本号为v3.0.5

truffle的使用并编译和发布合约，交互合约

用truffle创建项目

$ mkdir -p /home/liqing/Documents/Ethereum/truffle/workspace/test1

初始化框架

进入到工程根目录下(现在应该是个空目录)，并执行下述命令。
$ cd /home/liqing/Documents/Ethereum/truffle/workspace/test1

$ truffle init

正确执行后，我们将得到下面这样的目录结构：

目录结构简单说明如下：

  • contract/ - Truffle默认的合约文件存放地址。

  • migrations/ - 存放发布脚本文件

  • test/ - 用来测试应用和合约的测试文件

  • truffle.js - Truffle的配置文件

新建合约

  • 删除./contract目录下的自带demo合约，切记不要删除./contract/Migrations.sol合约，它是Truffle用来帮助部署的。

    $ cd /Users/admin/develop/blockchain_workspace/truffleTest/contracts

  • 在./contract目录下创建一个自己的合约文Greeter.sol。

    pragma solidity ^0.4.0;
contract Greeter
{
address creator;
string greeting;
function Greeter(string _greeting) public
{
creator = msg.sender;
greeting = _greeting;
}
function greet() constant returns (string)
{
return greeting;
}
function setGreeting(string _newgreeting)
{
greeting = _newgreeting;
}
/**********
Standard kill() function to recover funds
**********/
function kill()
{
if (msg.sender == creator)
suicide(creator); // kills this contract and sends remaining funds back to creator
}
}

修改发布脚本

将./migrations/2_deploy_contracts.js（如果没有新建一个，不要动1_xxx.js文件，文件是按照文件名编号顺序执行的）下的内容由：

module.exports = function(deployer) 
{
    deployer.deploy(ConvertLib);
    deployer.autolink();
    deployer.deploy(MetaCoin);
};

修改为：

var Greeter = artifacts.require("./Greeter.sol");
module.exports = function(deployer) {
    deployer.deploy(Greeter,"hellow world!");
};

目的是去掉原有自带的Demo部署流程，修改为要部署的合约。修改完后，记得保存，不然发布时会报错，找不到相关合约。

编译

进入到工程根目录下，进行编译：

$ truffle compile
Compiling Greeter.sol...
Writing artifacts to ./build/contracts

部署合约

部署合约需要有启动的节点，可以启动你的私链，也可以启动测试客户端testrpc

  • 启动你的客户端

启动之前安装好的EthereumJS RPC客户端。刚才如果启动了就不用管这步了。

  • 部署合约(migrate)

修改 truffle.js 的网络连接信息如下：

module.exports = {
    networks: {
        development: {
            host: "localhost",
            port: 8545,
            network_id: "*" // 匹配任何network id
        }
    }
};

部署合约到网络上：

$ truffle migrate

Running migration: 2_deploy_contracts.js
Deploying Greeter...
Greeter: 0xe66038995cf64b14da96d26cbe1c96d30dec0e95
Saving successful migration to network...
Saving artifacts...

备注

 如果报错出现了一些你之前编译部署过，但你已经不需要的合约，可以用truffle migrate –reset来重置部署，但可能根据情况，你需要更新移植版本号，来管理不同的版本，

运行合约

我们使用下面命令

truffle console

在控制台与合约交互，执行后显示。

truffle(development)>

这个破折号后面就可以输入东西了

我们输入

Greeter.deployed().then(function(instance){cc= instance;});

  • Greeter是合约的文件名也是合约名，合约的名字和合约的文件名要相同否则

    ReferenceError: test is not defined

  • 上面的命令意思是Greeter这个合约已经部署了，然后，将合约实例赋值给cc，那我们就可以通过cc来调用合约方法了。

  • 调用合约方法,输入cc. 如果不知道优什么方法，多按几次tab键 就会展示方法。

我们执行greet（）方法

truffle(development)> cc.greet();

返回

'hellow world!'

  • 控制台内，可以通过声明变量来减少输入的内容，比如

    var g = Greeter.deployed();g.greet();

  • 退出用 .exit

##问题来了

问题1.我如果要部署两个合约怎么弄？

  • 1.把合约文件放在contracts文件夹下

  • 2.修改2_deploy_contracts.js 文件

    var Greeter = artifacts.require("./Greeter.sol");//第一个合约
var test = artifacts.require("./DemoTypes.sol");//第二个合约
module.exports = function(deployer) {
deployer.deploy(Greeter,"hellow world!");
deployer.deploy(test);
};

  • 3.执行编译truffle compile

  • 4.部署 truffle migrate 出现了问题

    exceeds block gas limit

    这是说你的合约执行话费多余了你当前设置的最高限度，那我们要修改这个limit

    打开truffle.js 添加

    gas:4712388, //Gas limit used for deploys. Default is web3.eth.getBlock(“pending”).gasLimit==4712388.
gasPrice:18000000000, //Gas price used for deploys. Default is 100000000000 (100 Shannon).

    文件变为

    module.exports = {
  // See <http://truffleframework.com/docs/advanced/configuration>
  // to customize your Truffle configuration!
   networks: {
    development: {
    host: "localhost",
    port: 8545,
        gas:4712388, //Gas limit used for deploys. Default is web3.eth.getBlock(“pending”).gasLimit==4712388.
    gasPrice:18000000000, //Gas price used for deploys. Default is 100000000000 (100 Shannon).

    network_id: "*" // 匹配任何network id
    }
}
};

    我们重新设置了gas花费上限

    这样就可以了。后面就没有问题了。

问题2：为什么我的合约部署一直停在那里？

遇到这个问题，多半是你没有用testrcp，换成了自己的私链，而和testrcp相比，testrcp是自动挖矿的，如你开启了自己的私链，但没有挖矿，那么你的合约部署是不会执行的，所以解决方法就是，开启你私链的挖矿。miner.start()
