智能合约 solidity基础

发表于 2017-11-20   |    阅读次数

引

在以太坊-私有链的安装，启动，操作一文中，我们建立了一个自己的链，运行了一些命令，简单的操作。我们知道，以太坊和比特币区别之一是可以运行智能合约。那什么是智能合约呢？

智能合约

智能合约是运行在以太坊虚拟机（EVM）上的一段代码。合约可以完成一些交易，查询等功能。

为什么要写智能合约

智能合约可以在满足条件的情况下，自动执行代码，并完成合约内容。不受第三方干扰。比如，保险公司一张保单，如果这保单运行在以太坊系统上，当被保人达到一定赔付要求，保单就会自动运行检查和赔付功能。不需要被保人繁琐的提交证明。而且执行过程中不会被第三方打扰，做到公平，公正。

开发语言-solidity

作为智能合约的主要官方推荐开发语言。它的编译环境Remix 在一个在线的网页上。地址是https://ethereum.github.io/browser-solidity/#optimize=false&version=soljson-v0.4.21+commit.dfe3193c.js在这里可以编辑开发智能合约。

Remix简介

reminx是solidity编写智能合约的在线ide，检查代码的正确性并执行，是在remix所在的以太坊执行这段合约。

  • 1.代码区。编写合约。
  • 2.手动和自动编译按钮，自动编译选中后，每次改变代码，自动编译。
  • 3.当前编译的合约名字，点击Details显示合约相关信息，比如abi等。
  • 4.在1的下面，是执行后的输出区域。

右边切换到run的tab，

  • Environment，运行的环境，
  • Account 提供的测试用户。需要注意的是，当你点击create时，此栏选中的用户就是合约创建者，拥有者，msg.sender 也是它，但当你create完毕后，执行某个方法前，重新选了用户，那么这个msg.sender就是你当前选中的用户，但合约所有者不变，还是create时的那个用户。切记
  • Gas Limit 合约话费上限
  • value 如果你写的交易是需要话费gas的，那这里写gas的数量
  • 当前运行的合约

  • Create 点击此按钮时，上面Account中选择的用户则是此合约的创建用户。左边的输入框是当你的合约的构造函数中输入需要参数，那你点击create时需要输入参数。

  • 2 是此合约的地址，复制此地址填入到1的位置，点击at address，就会产生和点击create一模一样的结果。

  • 3 是点击create后产生的结果，就是你的合约中的方法，红色的是没有输出值的，就是结果不会显示在这里。灰色的就是执行完毕后会显示在这里，比如chairperson这个返回了当前合约的创建人。

solidity语法摘要

Solidity文件结构

一个合约文件可以包含任意数量的合约定义。

pragma

使用哪个版本的solidity编译,写在合约的开头。

版本指令应用举例：

pragma solidity ^0.4.0;

只能用0.4.0到0.5.0之间版本的编译器编译

第二个编译条件可以通过使用^添加

pragma solidity ^0.4.0^;

可以用0.4.0之后所有版本的编译器编译

更复杂的控制编译源码的编译器版本的表达式，可以遵照npm的规则设计。

源文件引用

Solidity引用源文件的方式与JavaScript（ES6及以后）类似，但是Solidity中没有“default export”。

  • 引入到全局变量空间：
    import “filename”;

把“filename”中所有变量全部导入到当前文件的全局变量空间（实际上是整个文件都导过来了，包括这个文件导入的其他文件）此用法与ES6的使用方式不同，但在Solidity中是向后兼容的。

  • 引入到自定义命名空间：
import * as symbolName from “filename";

把“filename”中所有变量全部导入到全局变量symbolName中并作为其成员。

import {symbol1 as alias, symbol2} from "filename";

把“filename”中symbol1和symbol2两个变量分别导入到全局变量alias和symbol2中，其中使用as重新命名symbol1。

文件路径也不必一定要映射到本地文件系统，也可以通过ipfs，http或git映射到网络文件。

举例

  • base.sol 

    contract BaseInclude {
    string mString;
    function BaseInclude(string pString){
        mString = pString;
    }
}

  • extends.sol

    import "./base.sol" ;//引入base.sol 文件

contract newChild is BaseInclude(){ //继承

    function newChild(string pStr) BaseInclude(pStr){

    }
    function main ()public constant returns (string){
        return mString;
    }
}

合约结构体

Solidity合约类似于面向对象语言的类。

每个合约都可以包含State Variables, Functions, Function Modifiers, Events, Structs Types 和 Enum Types声明。 合约也可以继承其他合约。

state variables 状态变量

  • 状态变量是永久存储在合约存储中的值。
pragma solidity ^0.4.0;
contract SimpleStorage {
    uint storedData; // 状态变量
    // ...
}

结构类型 struct

struct是可以包含几个变量的自定义类型，相当于java中的bean

pragma solidity ^0.4.0;
contract Ballot {
    struct Voter { // 结构体
        uint weight;
        bool voted;
        address delegate;
        uint vote;
    }
}

事件

事件是与EVM日志工具便捷接口。(需要补充)

移位操作

移位操作的结果是左操作数的类型。 表达式x << y等价于x 2 * y，x >> y等价于x / 2 y。 这意味着，移位负数符号延伸。 移位操作使用负数会引发运行时异常。
警告:
有符号整数类型的负值向右位移所产生的结果，与其他编程语言不同。 在“Solidity”中，将右移转换为除法，所以偏移的负值将向零舍入（截断）。 在其他编程语言中，负值的右移朝向负无穷大。

数据类型

值类型

就是传递值，而不是地址的数据类型，如果你把一个值类型作为参数传递给一个函数，那么在函数体内的任何对这个值类型的变量的操作，都不会影响到方法外的变量的值。

solidity的值类型的有：

bool：值的常是量是true和false。
int/uint：各种大小的有符号和无符号整数。
1.int是有符号整形，分别有int8，int16，int24，int32.。。。。一直到int256，步幅是8
2.uint是无符号整形，分别有uint8，uint16，uint24，uint32.。。。一直到uint256，步幅也是8
3.uint和int分别是uint256和int256的别名。

Address

地址：保存一个20个字节的值（Ethereum地址的大小）。 地址类型也有成员，作为所有合约的基础。

地址的操作

<=, <, ==, !=, >= 和 >  这是判断

地址成员 （地址属性）

  • balance

    myAddress.balance >= 10 判断地址余额是否大于等于0

  • transfer

    x.transfer(10) 给x地址 10个wei

如果x是合约地址，则其代码（更具体地说：其回退函数（如果存在））将与传送调用一起执行（这是EVM的限制，不能防止）。 如果执行任务无法运行，或以任何方式失败，则已经花费的以太币不会被退回，并且当前的合约将以异常方式停止。

  • send

    send是低级对等的转账。 如果执行失败，当前合约将不会以异常方式停止，但发送将返回false。

注意send和transfer的区别，send如果执行失败，不会抛出异常，但会返回false。transfer会报异常，程序终止，但没有返回值。

  • call, callcode and delegatecall

所有这些功能都是低级别的功能，应小心使用。 具体而言，任何未知的合约都可能是恶意的，如果你调用它，你将交出控制权，该合约可能又回调到你的合约，从而要改变你的状态变量。

固定大小的字节数组 byte
bytes1，bytes2，bytes3，…，bytes32。 byte是bytes1的别名

  • 字节数组操作：

    1.比较：<=，<，==，!=，>=，>（评估为bool）

    2.位运算符：＆，|，^（按位异或），〜（逐位否定），<<（左移），>>（右移）,移位操作符可以以任何整数类型作为右操作数（将返回左操作数的类型），用以表示要移位的位数。 移动负数将导致运行时异常。

    3.索引访问：如果x的类型为bytesi，则0 <= k <i的x [k]返回第k个字节（只读）

    4..length返回字节数组的固定长度（只读）

String 字符串类型

字符串文字用双引号或单引号（”foo”或’bar’）标注。 它们并不像C中那样以零结束; “foo”代表三个而不是四个字节。 与整数文字一样，它们的类型可以有所变化，它们可以隐式转换为bytes1，…，bytes32（如果它们适合）到字节和字符串。

字符串文字支持转义字符，例如\n，\xNN和\uNNNN。 \xNN取十六进制值并插入相应的字节，而\uNNNN则使用Unicode代码点并插入UTF-8序列

十六进制文字

Hexademical Literals以关键字hex为前缀，用双引号或单引号（hex”001122FF”）标注。 它们的内容必须是十六进制字符串，它们的值是这些值的二进制表示。

Hexademical Literals像字符串文字，具有相同的可转换性限制。

枚举类型

枚举可用于创建具有有限值集的自定义类型，在Solidity中，枚举是创建用户定义类型的一种方式。 它们可以显式转换为整数类型，也可以隐式转换。 显式转换在运行时检查值范围，失败会导致异常。 枚举需要至少一个成员

pragma solidity ^0.4.0;
contract test {
    enum ActionChoices { GoLeft, GoRight, GoStraight, SitStill }
    ActionChoices choice;
    ActionChoices constant defaultChoice = ActionChoices.GoStraight;
    function setGoStraight() {
        choice = ActionChoices.GoStraight;
    }
    // 由于枚举类型不是ABI的一部分，所以对于Solidity之外的所有事务，“getChoice”的签名将自动更改为“getChoice（）return（uint8）”）。 所使用的整数类型足够大以容纳所有枚举值，即如果你有更多值，则将使用`uint16`等等。
    function getChoice() returns (ActionChoices) {
        return choice;
    }
    function getDefaultChoice() returns (uint) {
        return uint(defaultChoice);
    }
}
函数类型

  • 函数是代码合约中的可执行单元。

    pragma solidity ^0.4.0;
contract SimpleAuction {
    function bid() payable { // 函数
        // ...
    }
}
引用类型

复杂类型，即不总是适合256位的类型，需要比其他值类型更仔细地处理。 由于复制它们相当昂贵，必须考虑我们是否希望将它们存储在内存中或存储（保存状态变量的位置）

数据位置 Data location （需要补充）
数组

数组可以是编译时固定大小，也可以是动态的。 对于存储阵列，元素类型可以是任意的（其他数组，映射或结构）。存储器阵列不能是映射，如果它作为公共可见函数的参数，则必须是ABI类型。
