solidity基础-函数

发表于 2017-11-21   |    阅读次数

函数类型

函数是代码合约中的可执行单元。

pragma solidity ^0.4.0;
contract SimpleAuction {
    function bid() payable { // 函数
        // ...
    }
}

函数前置修饰符 (需要补充)

modifiers 这个函数修饰符修饰函数，当这个函数被是哟用时，首先执行此函数，再执行调用此函数的函数。

举例：

pragma solidity ^0.4.11;
contract Purchase {
    address public seller;
    modifier onlySeller() { // Modifier 修饰的函数
        require(msg.sender == seller);
        _;
    }
    function abort() onlySeller returns (bool isSelf) { // 调用Modifier
        return isSelf
    }
}

abort函数使用了 onlySeller这个函数，在运行abort这个函数前，会先执行onlySelf方法的内容。

函数后置修饰符 (需要补充)

internal

private

public

whenNotPaused

payable 交易类型的函数，需要付gas

举例 

function transcationToSomeOne(address pAdderss) public payable{
        pAdderss.transfer(50);
}

此方法中的内容是向参数地址转50 wei个以太币，像这样的交易 需要声明方法名payable。

同时你要注意的是：

  • 在这个交易中，合约的创建人要转50wei的以太币给参数中的地址，方法是接受方调用transfer方法，而不是付款方。付款方默认是当前合约的创建者，就是执行create时，那个Account中选中的用户。如果你要付款50个以太币，就是

    pAdderss.transfer(50 ether); 量和单位之间加空格

  • 如果想执行这个交易，在value的部分你要输入需要消耗的gas，这个gas要大于你当前转账的50wei，就是至少是60wei。因为这种交易需额外的费用。
