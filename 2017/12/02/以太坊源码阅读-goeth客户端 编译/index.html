以太坊源码阅读-goeth客户端 编译

发表于 2017-12-02   |    阅读次数

以太坊客户端有好多版本，相对go版本比较稳定，拿来编译就能跑。

今天我们就来在Ubuntu下编译goeth客户端源码。

安装go

下载go安装包

wget https://www.golangtc.com/static/go/1.9.2/go1.9.2.linux-amd64.tar.gz

解压

sudo tar -C /usr/lib -xzf go1.9.linux-amd64.tar.gz

配置go环境变量

根据自己的工程名字如下配置环境变量：

使用 sudo vim /etc/profile 并添加下面的内容：

export GOROOT=/usr/lib/go
export GOBIN=$GOROOT/bin
export PATH=$GOROOT/bin:$PATH
export GOPATH=/home/{username} /{projectname}

/etc/profile:在登录时,操作系统定制用户环境时使用的第一个文件,此文件为系统的每个用户设置环境信息,当用户第一次登录时,该文件被执行。

以上修改完之后需要重启使得环境变量生效，或者在终端输入命令： 

source .bashrc

来更新go的安装目录

安装好之后可以在终端输入以下命令：

go env

如果安装成功则得到go的环境信息，否则安装失败

安装 C 编译器

apt install -y build-essential

工作目录的创建

Go代码必须放在工作空间内。它其实就是一个目录，其中包含三个子目录：

src 目录包含Go的源文件，它们被组织成包（每个目录都对应一个包），
pkg 目录包含包对象，
bin 目录包含可执行命令。
go 工具用于构建源码包，并将其生成的二进制文件安装到 pkg 和 bin 目录中。

src 子目录通常包会含多种版本控制的代码仓库（例如Git或Mercurial）， 以此来跟踪一个或多个源码包的开发。

例如GOPATH路径为/home/admin/project, 以下为GOPATH目录下的子目录结构

- bin
- pkg
- src
  - github.com
    - ethereum
      - go-ethereum
      -

注意 源码的路径非常深，go-ethereum 的绝对路径为/home/admin/project/src/github.com/ethereum/go-ethereum

所以 /home/admin/project/src/github.com/ethereum/都需要自己创建对应的目录。

下载源码

上述的go-ethereum 并不用自己新建，只需要切换到/home/admin/project/src/github.com/ethereum/目录下（如果没有，就需要自己新建了），然后执行

git clone git@github.com:ethereum/go-ethereum.git

编译生成可执行文件

cmd目录下的每一个文件都是一个单独的可编译的程序（都包含main.go文件）
通过以下命令，可以编译全部文件：

admin@ubuntu:~/project/src/github.com/ethereum/go-ethereum$ make  
Done building.
Run "build/bin/geth" to launch geth.

最后生成的可执行文件为 build/bin/geth ，可以正常启动geth, 也可以添加需要的参数（比如 —datadir, -rpcport之类的）。可以发现，这里编译生成的geth和通过apt-get安装的作用是一样的。
